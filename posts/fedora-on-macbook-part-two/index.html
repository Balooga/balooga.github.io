Installing Fedora 32 on a mid-2009 Macbook Pro - Part Two; i3wm

September 2, 2020 7-minute read
These posts document an installation of Fedora 32 onto a mid-2009 15" Macbook Pro. It is in two parts; Part One describes the initial installation of Fedora and necessary updates to detect the internal WiFi hardware, and Part Two describes configuring the window manager i3wm. As is usual with Linux installations, forewarned is forearmed. Be prepared for the lack of internal WiFi capability in the base Fedora install. And, if you use the i3 Window Manager, there is additional work to allow the MacBook to return to active mode from standby when the lid is closed and subsequently re-opened.

Installing i3wm

I’m using i3wm[1], a fast but bare-bones "tiling" Window Manager.

1

$ sudo yum -y install i3 i3status dmenu feh rofi terminator i3lock

Log into i3wm and set your mod key. i3wm then creates the ~/.config/i3/config configuration file.

Niceties in the Xfce4 installation image

i3wm requires additional utilities and applets to perform network management and configuration, bluetooth pairing, power management, and mouse/trackpad and brightness controls for screen and keyboard. Installing the Xfce[2] Fedora "Spins" Live Image — as I described in [part-one] — provides much of this functionality in a way that integrates well with i3wm.

Use the bluetooth applet to display the bluetooth icon in the i3 status bar.

Add the following to ~/.config/i3/config

1

exec --no-startup-id /usr/bin/blueman-applet                # Bluetooth Manager

Use the Xfce the power manager applet to control screen and keyboard brightness. The power management icon is displayed in the the i3 status bar.

Add the following to .~/.config/i3/config

exec --no-startup-id /usr/bin/xfce4-power-manager           # Power Manager. For
                                                            # controlling screen
                                                            # and keyboard
                                                            # brightness, etc.

NVidia driver support for additional display

The Nouvea drivers[3] seem to work well for screen modes up to the MacBook Pro’s native screen resolution of 1440x900. However monitors having a resolution greater than this seem to exhibit flickering and tearing. To fix this, install the correct version of the NVidia graphics drivers.[4]

Install all updates, and add the RPMFusion repo that contains the nvidia drivers.
1
2

$ sudo dnf update
$ sudo dnf install https://download1.rpmfusion.org/free/fedora/rpmfusion-free-release-$(rpm -E %fedora).noarch.rpm https://download1.rpmfusion.org/nonfree/fedora/rpmfusion-nonfree-release-$(rpm -E %fedora).noarch.rpm
Use lspci to list the model of graphics hardware.
1

$ lspci -vnn | grep VGA
The Macbook Pro 2009 15" requires the oldest nvidia-340xx series drivers.
1
2
3
4

$ sudo dnf install xorg-x11-drv-nvidia-340xx akmod-nvidia-340xx

# sudo dnf install akmod-nvidia
# sudo dnf install xorg-x11-drv-nvidia-390xx akmod-nvidia-390xx

Swap CAPS and Control

Log out and back into i3wm for these settings to take effect. Or use something like the HHKB[5] keybaord and skip this section.

Add the following to ~/.config/i3/config

# Swap CAPS_LOCK and R_CTRL
# To get the current key mappings
# $ xmodmap -pke
exec --no-startup-id /usr/bin/xmodmap /.Xmodmap

1

bindsym $mod+Control+l exec i3lock

Setting the background image

The following command creates a ~/.fehbg file to run at i3wm startup
1

$ feh --bg-scale <path_to_background_image>
Add the following to ~/.config/i3/config
1
2

# Set the background image
exec --no-startup-id ~/.fehbg

        -show drun -sidebar-mode -columns 3 -font 'Noto Sans 8'

bindsym $mod+Shift+d exec dmenu_run

bindsym $mod+Shift+3 move container to workspace $ws3
bindsym $mod+Shift+3 move container to workspace $ws4
# ...

# Create a specific workspace for Spotify
