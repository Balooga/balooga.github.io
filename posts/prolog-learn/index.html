Prolog Learnings: Unification

December 9, 2020 3-minute read
The concept of Unification is an underpinning of Prolog and why functions in Prolog are called "predicates". Because unlike a function that can only evaluate its inputs to create an output, predicates can derive possible inputs from an output. In idiomatic Prolog, predicates work both ways.

Most languages assign values to variables. For example, the statement int x = 0; causes the value zero to be assigned to x, resulting in x having a value of zero. Prolog performs "Unification" which resembles assignment but is more powerful. The 'clause' below (Prolog uses the terms clauses, predicates and rules) results in X being assigned a value of zero (variables in Prolog must start with an uppercase letter), as a result of Prolog evaulating the clause to "true" by "Unifying" (make the same) the terms on the left of the = with the terms on the right. In this example, Prolog unifies left and right by making X equal zero.

Prolog
1
2

?- X = 0.
X = 0.

If X is already assigned a value then any later attempts to unify to a different value will fail. This behaviour is different to "iterative" languages where a value assigned to a variable can change.

Prolog
1
2

?- X = 1, X = 0.
false

?- reverse([1,2,3], Output).
Output = [3,2,1].

?- reverse(Input, [3,2,1]).
Input = [1,2,3].

?- reverse([1,2,3], [3,2,1]).
true.

When a predicate is not provided an input and output, Prolog will attempt to unify by creating anonymous variables as needed (anonymous variables are prefixed with an underscore). In the example below, the semicolon (;) prompts Prolog to use backtracking to attempt to find alternate inputs that unify to true.
