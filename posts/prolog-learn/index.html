Unification versus Assignment

Every language I’ve used up to now assigns values to variables. For example;

1

int x = 0;

The statement above causes zero to be assigned to x, resulting in x having the value of zero. Prolog uses "Unification" which on the surface resembles assignment but is more powerful.

1
2

?- X = 0.
X = 0

The 'clause' above (Prolog uses the terms clauses, predicates and rules) results in X having the value of zero through a process called Unification (variables in Prolog must start with an uppercase letter.) Prolog attempts to make the clause evaulate to "true" by "Unifying" (make the same) the terms on the left of the = with the terms on the right. In this example, Prolog unifies the left and right sides by making X equal zero, which results in the left and the right evaluating to true.

Unification works both ways as the following examples show;

?- [1,2,3] = [1,2,3].
true.

?- reverse([1,2,3], Output).
Output = [3,2,1].

?- reverse(Input, [3,2,1]).
Input = [1,2,3].

?- reverse([1,2,3], [3,2,1]).
true.

This concept of Unification is an underpinning of Prolog and is why in Prolog functions are called "predicates". Because unlike a function that only evaluates inputs to create an output, predicates also evaluate an output to create possible inputs. In idiomatic Prolog, predicates work both ways. Think of predicates as a way to ask Prolog "What are the possible inputs to drive this output?"

Even when not provided an input and output, Prolog will still attempt to unify by creating anonymous variables as needed (anonymous variables are prefixed with an underscore). In the example below, the semicolon (;) prompts Prolog to continue to find alternates that unify to true by using backtracking.
